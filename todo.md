### Technical-Flavor

| Stufe | Beschreibung |
|-------|-------------|
| Bereits erledigt | kein Aufwand nötig |
| Trivial | wenige Minuten, kein Risiko |
| Sehr klein | gut verstanden (Referenzaufgabe) |
| Kleine Aufgabe | etwas mehr als eine 1 |
| Klein bis mittel | einfach aber braucht etwas Überlegung |
| Mittlerer Aufwand | moderate Komplexität |
| Groß | erhebliche Komplexität, Aufteilen erwägen |
| Sehr groß | hohe Komplexität und Unsicherheit, sollte aufgeteilt werden |
| Extrem groß | zu viel für einen einzelnen Sprint |
| Epic-Niveau | muss zerlegt werden |
| Riesig | ein Projekt für sich, muss zerlegt werden |

### Software-Flavor

| Begriff | Beschreibung |
|---------|-------------|
| Noop | keine Operation, ein No-Op |
| Config-Änderung | Flag umschalten, Konstante ändern |
| Einzeiler-Fix | eine einzelne, gut verstandene Codeänderung |
| Kleiner Bugfix | finden, patchen, testen |
| Einfaches Feature | ein Formular, ein Button, ein neuer Endpunkt |
| Feature mit Tests | echte Feature-Arbeit mit Grenzfällen |
| Mehrere Komponenten | betrifft mehrere Dateien/Module, braucht Koordination |
| Subsystem | neues Subsystem oder größerer Umbau eines bestehenden |
| Architekturänderung | strukturelle Änderung quer durch die Codebasis |
| Plattform-Migration | Umzug auf ein neues Framework/Plattform |
| Komplett-Neubau | von vorne anfangen, bist du sicher? |
